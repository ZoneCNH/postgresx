# postgresx

`postgresx` is the PostgreSQL runtime foundation library for Go services.

It provides a small infrastructure layer around PostgreSQL: pgx pool lifecycle,
configuration validation, sqlc-compatible execution, explicit transactions,
retry policy, migration runner, error normalization, health checks, pool
statistics, optional metrics/tracing adapters, DSN masking, and integration test
helpers.

It is not an ORM, does not own business schema, and does not depend on any consumer application.

## Install

```sh
go get github.com/ZoneCNH/postgresx
```

## Client

Callers provide explicit configuration. Secret loading belongs in the
application boundary, not in `pkg/postgresx`.

```go
password := foundationx.NewSecretString(driverPasswordFromYourConfigBoundary)
client, err := postgresx.Open(ctx, postgresx.Config{
    Host:            "127.0.0.1",
    Port:            5432,
    Database:        "app",
    User:            "app",
    Password:        password,
    SSLMode:         "disable",
    ApplicationName: "postgresx-example",
    MaxOpenConns:    16,
})
if err != nil {
    return err
}
defer client.Close(ctx)
```

`postgresx` owns the pgx pool lifecycle behind `Client`. Callers pass explicit
configuration and optional logger, metrics, and clock options. Metrics and
tracing adapters emit only when explicitly configured.

## sqlc

`postgresx.Queryer` matches the narrow query interface generated by sqlc.

```go
q := generated.New(client.Queryer())
```

Inside a transaction:

```go
err := client.WithTx(ctx, func(ctx context.Context, tx postgresx.Tx) error {
    q := generated.New(tx)
    return q.UpdateCursor(ctx, params)
})
```

Use `WithTxOptions(ctx, postgresx.TxOptions{...}, fn)` when transaction options
are required.

## Migrations

Applications own their migration files and schema. `postgresx` only runs SQL
provided by the caller.

```go
runner := postgresx.NewMigrationRunner(client)
return runner.Up(ctx, appMigrationSource)
```

## Secret Masking

Callers supply secrets explicitly through `Config.Password`. Use
`Config.RedactedDSN()` or `Config.Sanitize()` before logging configuration or
connection failures.

```go
log.Printf("postgres dsn=%s", cfg.RedactedDSN())
```

## Verification

```sh
GOWORK=off make test
GOWORK=off make vet
GOWORK=off make race
GOWORK=off make contracts
GOWORK=off VERSION=v1.0.0 make release-check
```

`POSTGRES_TEST_DSN` or `POSTGRESX_INTEGRATION_DSN` is required only for live
PostgreSQL integration when Docker is unavailable. Release mode requires live
integration with `POSTGRESX_REQUIRE_INTEGRATION=1`; inject those environment
variables from the caller or CI environment and do not commit DSNs.

The local release gate is `L2-T3 / 85`: unit, contract, integration, chaos,
benchmark smoke, and downstream compile smoke must pass before evidence is
generated. `release_allowed=true` is local to this gate;
`factory_grade_allowed=false` remains until external CI, production soak, and
real consumer adoption evidence are available.

`make release-check` regenerates current-branch evidence from `HEAD`.
`make release-evidence-check` verifies the immutable published tag manifest.
For the existing `v1.0.0` tag, that immutable check remains blocked because the
restored snapshot source commit `9eaf770` is outside the current `HEAD` and tag
ancestry; do not overwrite or retag `v1.0.0` with post-tag evidence without an
explicit release-history decision.
